# PM BY MADI — платформа перманентного макияжа

Полноценный сайт студии перманентного макияжа с маркетинговой публичной частью и административной панелью для управления контентом.

## Основной стек

- Next.js 15 (App Router) + TypeScript
- Tailwind CSS + shadcn/ui
- Prisma ORM + SQLite (готовность к PostgreSQL)
- NextAuth (credentials)
- React Hook Form + Zod
- ESLint + Prettier

## Быстрый старт

```bash
npm install
npx prisma migrate dev --name init
npm run seed
npm run dev
```

Откройте `http://localhost:3000`. Админ-панель доступна по адресу `http://localhost:3000/admin`.

### Доступы по умолчанию

```
Email:    admin@example.com
Password: admin12345
```

## Возможности

### Публичная часть
- Главная страница с hero-блоком, преимуществами, портфолио, отзывами и блогом.
- Страницы услуг и карточки с детальной информацией, ценами, подготовкой и уходом.
- Портфолио, отзывы, акции, блог.
- О мастере, FAQ, юридические страницы (политика, оферта).
- Контакты с интерактивной формой записи и отправкой вопросов.
- SEO: sitemap, robots, динамические метаданные из БД.

### Админ-панель `/admin`
- Dashboard с ключевыми показателями и свежими заявками.
- Управление категориями услуг, услугами и прайсами.
- CRUD для портфолио, отзывов, акций, блога.
- Управление страницами, FAQ, настройками бренда и контактами.
- Медиа-менеджер с загрузкой файлов на `/public/uploads/yyyy/mm`.
- Просмотр и обновление статусов заявок.

## Структура проекта

- `app/` — страницы, маршруты API, административные разделы.
- `components/` — UI-компоненты (включая библиотеку shadcn/ui), формы, элементы навигации.
- `actions/` — серверные экшены для CRUD-операций.
- `lib/` — Prisma, настройки, вспомогательные функции и запросы.
- `prisma/` — схема БД и сиды.
- `public/uploads` — директория для загруженных изображений.

npm run lint     # проверка ESLint

## Переменные окружения

Скопируйте `.env.example` в `.env` и при необходимости измените значения:

```
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=changeme
DATABASE_URL=file:./dev.db
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=admin12345
```

Для PostgreSQL замените `DATABASE_URL` на строку подключения и выполните миграцию.

## Деплой

Проект готов к публикации на Vercel или любой платформе для Next.js:
1. Выполните `npm run build` локально, убедитесь, что сборка проходит.
2. Настройте переменные окружения в панели деплоя (см. раздел выше).
3. При необходимости выполните `prisma migrate deploy` и `npm run seed` на сервере.

## Настройка под студию

Все данные редактируются через админ-панель. При необходимости можно обновить стартовые сиды в файле `prisma/seed.ts`.
